Hello everyone.

Today, we are presenting our project, the Open Library Book System, a full-stack web application developed using React, Flask, and PostgreSQL.

The idea behind this project came from a common problem faced by many readers: while people enjoy reading books, there isn't always a simple and centralized way to manage book collections, ratings, and personal reviews. Existing solutions can be overly complex or focused only on purchasing books rather than tracking reading experiences.

Our solution addresses this by providing a platform where users can easily add books to a collection, browse existing books, submit reviews and ratings, update information when needed, and remove books or reviews that are no longer relevant.

From a technical perspective, the application follows a modern full-stack architecture. The frontend was built using React to create an interactive and responsive user interface, while the backend was developed using Flask to provide RESTful API services. PostgreSQL was used as the database to ensure reliable and structured data storage.

One of the key features of the application is the implementation of full CRUD operations for both books and reviews. The system also establishes a one-to-many relationship where a single book can have multiple reviews associated with it.

The frontend communicates with the backend through API calls, allowing users to interact with data seamlessly without directly accessing the database.

This project allowed us to gain practical experience in frontend development, backend API design, database management, version control using Git and GitHub, deployment, and collaborative software development.

In conclusion, the Open Library Book System demonstrates the core principles of modern full-stack development while providing a practical and user-friendly solution for managing books and reviews.
